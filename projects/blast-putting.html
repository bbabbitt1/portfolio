Blast Motion Putting Pipeline

A serverless ETL pipeline that extracts putting stroke data from Blast Motion sensors via the Blast Connect API, loads it into Azure SQL, and refreshes a Power BI dashboard. Deployed as an Azure Function with timer and HTTP triggers, orchestrated via Power Automate for on-demand mobile sync.

Architecture

Azure Function Triggers

The pipeline runs as a serverless Azure Function with two triggers: a nightly timer and an on-demand HTTP endpoint secured with a function key.

function_app.pyPython
import azure.functions as func

app = func.FunctionApp()

# Nightly sync at 5 AM UTC
@app.timer_trigger(schedule="0 0 5 * * *", arg_name="timer")
def blast_timer_trigger(timer: func.TimerRequest) -> None:
    from main import run_pipeline
    result = run_pipeline()

# On-demand via POST (secured with function key)
@app.route(route="run-pipeline", auth_level=func.AuthLevel.FUNCTION)
def blast_http_trigger(req: func.HttpRequest) -> func.HttpResponse:
    from main import run_pipeline
    result = run_pipeline()
    return func.HttpResponse(f"Pipeline complete: {result}")

Pipeline Runner

The pipeline authenticates via CSRF token, fetches new sessions using a high watermark, cleans the data, and upserts into Azure SQL — returning a summary for the Power Automate notification.

main.pyPython
def run_pipeline():
    logger.info("Starting Blast Motion pipeline")
    try:
        session = requests.Session()

        # Step 1-2: Authenticate
        token = get_csrf_token(session)
        login(session, token)

        # Step 3: Fetch all sessions
        sessions_list, sessions_data = get_sessions(session)

        # Step 4: Incremental load via high watermark
        engine = get_engine()
        max_session_id = int(get_high_watermark(engine))

        # Step 5: Only process new data
        new_sessions = [s for s in sessions_list if int(s['id']) > max_session_id]

        if not new_sessions:
            return "No new sessions found"

        # Step 6-7: Extract & transform
        all_putts = get_putts(session, new_sessions)
        df_putts = clean_putts_df(pd.DataFrame(all_putts))
        sessions_df = clean_sessions_df(pd.DataFrame(new_sessions))

        # Step 8: Atomic load to Azure SQL
        with engine.begin() as conn:
            upsert_sessions(sessions_df, engine)
            upsert_putts(df_putts, engine)
            return f"Synced {len(new_sessions)} sessions, {len(df_putts)} putts"

Incremental Loading

The pipeline uses a high watermark query to avoid reprocessing data. Only sessions with IDs greater than the current max are fetched from the API.

get_high_watermark.sqlSQL
SELECT
    COALESCE(max(blast_session_id), 0)
FROM src.blast_sessions;

Idempotent Upserts

Uses SQL Server's IF NOT EXISTS pattern for safe, repeatable writes. If the pipeline runs twice, no duplicates are created.

utils.pyPython
def upsert_sessions(df: pd.DataFrame, engine) -> None:
    with engine.begin() as conn:
        for _, row in df.iterrows():
            conn.execute(text("""
                IF NOT EXISTS (SELECT 1 FROM src.blast_sessions
                    WHERE blast_session_id = :blast_session_id)
                INSERT INTO src.blast_sessions
                    (blast_session_id, session_type_id, session_name)
                VALUES (:blast_session_id, :session_type_id, :session_name)
            """), row.to_dict())
        logger.info(f"Upserted {len(df)} sessions")

Resilient HTTP Client

The pipeline includes exponential backoff with jitter for transient failures, and fails fast on auth or rate-limit errors.

utils.pyPython
def get_with_retry(session, url, params=None, retries=MAX_RETRIES, base_wait=BASE_WAIT):
    for attempt in range(retries):
        try:
            response = session.get(url, params=params, timeout=TIMEOUT)
            if response.status_code == 200:
                return response
            elif response.status_code == 401:
                raise BlastAuthError("Unauthorized")
            elif response.status_code == 429:
                raise BlastRateLimitError("Rate limited")
            elif response.status_code >= 500:
                raise BlastServerError(f"Server error {response.status_code}")
        except (ConnectionError, Timeout, BlastServerError) as e:
            if attempt < retries - 1:
                # Exponential backoff + jitter
                wait = (base_wait ** attempt) + random.uniform(0, 1)
                time.sleep(wait)
            continue
    raise BlastAPIError(f"All {retries} attempts failed")

Mobile Orchestration

Power Automate provides a one-tap mobile workflow: trigger the pipeline, refresh the Power BI dataset, and receive a push notification with sync results.

Button tap → HTTP POST to Azure Function → Pipeline syncs putts → Power BI dataset refresh → Push notification: "Pipeline complete: Synced 3 sessions, 15 putts"

Dashboard Output

The Power BI dashboard surfaces 9 key putting metrics with trend lines, target ranges, and status indicators across configurable putt windows (25, 50, 100, or all).

Database Schema

Azure SQL Database with a src schema for raw sensor data. Sessions and putts are linked by session ID, with 11 stroke metrics captured per putt.

Strokes Gained Calculator

A Python desktop application for tracking golf rounds shot-by-shot and computing real-time strokes gained metrics against PGA Tour benchmarks. Features a star schema SQLite database, live calculation feedback, and Matplotlib analytics visualizations.

Architecture

The app uses an in-memory cache pattern — no database writes occur until the round is complete. All shots are persisted atomically in a single transaction on the summary screen.

Strokes Gained Calculation

The core algorithm computes strokes gained in real-time as each shot is entered. It looks up PGA Tour average strokes-to-hole from both the starting and ending positions, then applies the formula: SG = TourAvg(start) - (1 + TourAvg(end)).

shot_entry.pyPython
def calculate_strokes_gained():
    """Compute SG live — no DB write, instant UI feedback."""
    surf_start = surface_start_dd.get()
    surf_end = surface_end_dd.get()
    dist_start = float(distance_start_entry.get())

    conn = sqlite3.connect(DB_PATH)
    c = conn.cursor()

    # Lookup starting position baseline
    c.execute(
        "SELECT TourAvg FROM DimAvg WHERE Surface=? AND Distance=?",
        (surf_start, int(dist_start))
    )
    start_row = c.fetchone()

    # Lookup ending position (or 0 if holed out)
    if surf_end == "Hole":
        end_val = 0
    else:
        dist_end = float(distance_end_entry.get())
        c.execute(
            "SELECT TourAvg FROM DimAvg WHERE Surface=? AND Distance=?",
            (surf_end, int(dist_end))
        )
        end_val = c.fetchone()[0]

    # Core formula: SG = TourAvg(start) - (1 + TourAvg(end))
    sg_val = start_row[0] - (1 + end_val)
    if penalty_var.get():
        sg_val -= 1

    # Color-coded feedback: green = gained, red = lost
    color = GREEN_TEXT if sg_val > 0 else RED_TEXT if sg_val < 0 else TEXT_COLOR
    sg_label.config(text=f"Strokes Gained: {sg_val:+.2f}", fg=color)
    conn.close()

Atomic Round Persistence

When the round is complete, all shots are saved in a single atomic transaction — dimension record first (DimRound), then all fact records (FactShots) linked by the generated round ID.

round_summary.pyPython
def save_round():
    """Atomically persist a complete round."""
    conn = sqlite3.connect(DB_PATH)
    c = conn.cursor()

    # Insert dimension record
    c.execute("""
        INSERT INTO DimRound
            (PlayerID, CoursePlayed, RoundDate, HolesPlayed, TeePreference)
        VALUES (?, ?, date('now'), ?, 'Tips')
    """, (shots[0]["PlayerID"], round_info["CoursePlayed"], len(holes)))

    round_id = c.lastrowid  # Auto-generated FK

    # Batch insert all shots
    for s in shots:
        c.execute("""
            INSERT INTO FactShots
                (PlayerID, RoundID, Hole, Par, HoleResult,
                 Category, SurfaceStart, DistanceStart,
                 SurfaceEnd, DistanceEnd, ClubUsed,
                 ShotShape, Penalty, StrokesGained)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
        """, (s["PlayerID"], round_id, s["Hole"], s.get("Par"),
              s.get("HoleResult"), s["Category"], s["SurfaceStart"],
              s["DistanceStart"], s["SurfaceEnd"], s["DistanceEnd"],
              s["ClubUsed"], s["ShotShape"], s["Penalty"],
              s["StrokesGained"]))

    conn.commit()  # Atomic transaction
    conn.close()

Round Analytics

After completing a round, the summary screen computes standard golf KPIs — Greens in Regulation, Fairways in Regulation, strokes gained by category — and renders a Matplotlib breakdown chart.

round_summary.pyPython
# Aggregate round data into KPIs
holes = {}
for shot in shots:
    holes.setdefault(shot["Hole"], []).append(shot)

for hole, shots_list in holes.items():
    par = int(shots_list[0].get("Par", 0))
    total_par += par
    strokes = len(shots_list) + sum(s.get("Penalty", 0) for s in shots_list)
    total_strokes += strokes

    # Green in Regulation: reached green in (par - 2) or fewer
    strokes_to_green = next(
        (i + 1 for i, s in enumerate(shots_list)
         if s.get("SurfaceEnd") in ("Green", "Hole")),
        None
    )
    if strokes_to_green and strokes_to_green <= (par - 2):
        greens_hit += 1

gir_pct = (greens_hit / holes_played * 100) if holes_played else 0

Star Schema Design

The database follows a classic star schema pattern: dimension tables for players, rounds, and tour averages, with a central fact table for individual shots.

Database SchemaSQLite
-- Dimension: Players
CREATE TABLE DimPlayer (
    PlayerID    INTEGER PRIMARY KEY,
    PlayerName  TEXT    NOT NULL
);

-- Dimension: Rounds
CREATE TABLE DimRound (
    RoundID       INTEGER   PRIMARY KEY AUTOINCREMENT,
    PlayerID      INTEGER   NOT NULL REFERENCES DimPlayer(PlayerID),
    CoursePlayed  TEXT,
    RoundDate     TEXT,
    HolesPlayed   INTEGER
);

-- Dimension: PGA Tour Benchmarks (from CSV)
CREATE TABLE DimAvg (
    Surface   TEXT,
    Distance  INTEGER,
    TourAvg   REAL     -- Expected strokes to hole
);

-- Fact: Individual Shots
CREATE TABLE FactShots (
    ShotID         INTEGER   PRIMARY KEY AUTOINCREMENT,
    PlayerID       INTEGER   REFERENCES DimPlayer(PlayerID),
    RoundID        INTEGER   REFERENCES DimRound(RoundID),
    Hole           INTEGER,
    Category       TEXT,      -- Driving, Approach, Short Game, Putting
    SurfaceStart   TEXT,
    DistanceStart  REAL,
    SurfaceEnd     TEXT,
    DistanceEnd    REAL,
    StrokesGained  REAL
);

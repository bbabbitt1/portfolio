M-Query: PDF Folder ETL

A common Power Query scenario — ingesting a folder of PDF pay stubs, extracting structured tables, and cleaning them into an analysis-ready dataset. This walkthrough starts with a naive approach and refactors it into an optimized query using engineering best practices.

Power Query M Language PDF Ingestion ETL Performance

The Scenario

You have a folder of PDF pay stubs (PaySlip25/). Each file contains multiple pages of tables. You need to extract the data from Page 1 of every PDF, filter out header/summary rows, parse the pay-period date from the filename, and type the numeric columns.

Naive Approach

  • Expand all PDF pages, then filter to Page001
  • Expand data columns in a second pass
  • Carry binary Content through 5+ steps
  • Add an Index column that's never used downstream
  • Filter rows after full expansion

Optimized Approach

  • Direct record access — no expand for page selection
  • Row-level filtering inside transform — before combine
  • Binary Content dropped immediately
  • No unnecessary columns carried through the pipeline
  • Single Table.Combine at the end
1 Read the folder — drop binaries immediately

Folder.Files returns every column including the heavy binary Content. Select only Content and Name right away so the engine doesn't carry file attributes and metadata through every downstream step.

let // 1. Read folder — keep only what we need Source = Folder.Files("C:\PaySlips"), Files = Table.SelectColumns(Source, {"Content", "Name"}),
2 Extract metadata from the filename first

Parse the pay-period date from the filename before touching PDF content. This is a cheap string operation — do it while the table is still one row per file.

// 2. Extract week-end date from filename (cheap — do it early) AddDate = Table.AddColumn( Files, "WeekEndDate", each Text.BetweenDelimiters([Name], "tt_", ".pdf"), type text ),
3 Parse PDFs & filter in one transform — no expand

This is the key optimization. Instead of expanding Pdf.Tables into rows and then filtering to Page001, use direct record access with {[Id="Page001"]} to grab the exact page. Then filter out unwanted rows inside the same transform — before any data leaves the per-file scope.

// 3. Parse each PDF → grab Page001 → filter rows (no expand) Parsed = Table.AddColumn(AddDate, "Data", each let pages = Pdf.Tables([Content]), page1 = pages{[Id="Page001"]}[Data], excluded = {"Employee Name", "Employee Taxes", "Employer Taxes", "Tax", "Total"}, cleaned = Table.SelectRows(page1, each [Column1] <> null and not List.Contains(excluded, [Column1]) ) in cleaned ),
Why this matters: The naive approach expands every page of every PDF into rows, then filters down. With 50 PDFs × 3 pages each, that's 150 rows before filtering. Direct access means 50 rows — one per file — and zero wasted page expansions.
4 Drop binaries, then expand the cleaned data

Remove Content and Name before expanding — the binary column is no longer needed and Name has been replaced by WeekEndDate. Now the only expand is the pre-filtered, pre-cleaned page data.

// 4. Drop heavy columns, expand only cleaned data Trimmed = Table.SelectColumns(Parsed, {"Data", "WeekEndDate"}), Expanded = Table.ExpandTableColumn( Trimmed, "Data", {"Column1", "Column2", "Column3"} ),
5 Type columns in a single step

Apply all type conversions in one Table.TransformColumnTypes call at the end. Multiple type-change steps force the engine to re-scan the table each time.

// 5. Type all columns in one pass Typed = Table.TransformColumnTypes(Expanded, { {"Column1", type text}, {"Column2", type number}, {"Column3", type number}, {"WeekEndDate", type date} }) in Typed

Complete Optimized Query

let // Read folder — keep only Content + Name Source = Folder.Files("C:\PaySlips"), Files = Table.SelectColumns(Source, {"Content", "Name"}), // Extract pay-period date from filename before heavy work AddDate = Table.AddColumn( Files, "WeekEndDate", each Text.BetweenDelimiters([Name], "tt_", ".pdf"), type text ), // Parse each PDF → direct access to Page001 → row-level filter Parsed = Table.AddColumn(AddDate, "Data", each let pages = Pdf.Tables([Content]), page1 = pages{[Id="Page001"]}[Data], excluded = {"Employee Name", "Employee Taxes", "Employer Taxes", "Tax", "Total"}, cleaned = Table.SelectRows(page1, each [Column1] <> null and not List.Contains(excluded, [Column1]) ) in cleaned ), // Drop binary + filename, expand only cleaned page data Trimmed = Table.SelectColumns(Parsed, {"Data", "WeekEndDate"}), Expanded = Table.ExpandTableColumn( Trimmed, "Data", {"Column1", "Column2", "Column3"} ), // Type all columns in a single pass Typed = Table.TransformColumnTypes(Expanded, { {"Column1", type text}, {"Column2", type number}, {"Column3", type number}, {"WeekEndDate", type date} }) in Typed

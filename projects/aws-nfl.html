AWS NFL Analytics

End-to-end data pipeline that scrapes NFL player stats and Vegas betting lines from pro-football-reference.com, stages data in AWS S3 with partitioned storage, and transforms it through dbt models on Snowflake for analytics.

Architecture

Web Scraping Pipeline

The scraper navigates pro-football-reference.com, parsing HTML comments (where the site hides data tables), builds dynamic URLs for each player's game log, and streams results directly to S3.

Player_Data.ipynbPython
# Parse HTML comments — pro-football-reference hides tables here
soup = BeautifulSoup(page, 'lxml')
comments = soup.find_all(string=lambda text: isinstance(text, Comment))

roster_html = None
for comment in comments:
    if 'table' in comment and 'id="roster"' in comment:
        roster_html = comment
        break

if roster_html:
    df_roster = pd.read_html(roster_html, attrs={'id': 'roster'})[0]
    tree = html.fromstring(roster_html)
    player_links = tree.xpath(
        '//table[@id="roster"]//td[@data-stat="player"]/a/@href'
    )
    df_roster['Full_Player_URL'] = [
        f'https://www.pro-football-reference.com{href}/gamelog/{season}'
        if href else None
        for href in player_links
    ]

S3 Data Lake

Data is streamed to S3 using in-memory buffers (no temp files) with Hive-style partitioning by season, team, and position.

Player_Data.ipynbPython
# Stream DataFrame to S3 — no temp files

csv_bytes_buffer = io.BytesIO(csv_str_buffer.getvalue().encode("utf-8"))
csv_bytes_buffer.seek(0)

# Hive-style partitioning
s3_path = f'player_data/season={season}/position={position}/{season}_{position}_{player_name}_data.csv'
client.upload_fileobj(csv_bytes_buffer, BUCKET_NAME, s3_path)

# Respectful rate limiting
time.sleep(random.randint(6, 8))
The scraper processes 32 NFL teams across multiple seasons, building individual game logs for every QB, RB, WR, TE, and K. Rate limiting (6-8s between requests) prevents IP blocks.

Feature Engineering

Raw Vegas betting data is cleaned and enriched with computed features — game outcome classification, regular season vs. playoff detection (accounting for the 2021 schedule expansion), and score parsing.

Team_Data.ipynbPython
def clean_vegas_df(df):
    # Determine home/away from opponent formatting
    df['Home'] = df['Opp'].apply(lambda x: 0 if x.startswith('@') else 1)
    df['Opp'] = df['Opp'].apply(lambda x: x.strip('@'))

    # Parse score from result string: "W, 27-24"
    df['points_scored'] = df['Result'].apply(
        lambda x: x.split(', ')[1].split('-')[0]
    )
    df['points_allowed'] = df['Result'].apply(
        lambda x: x.split(', ')[1].split('-')[1]
    )

    # Handle 2021 schedule expansion (16 → 17 games)
    def game_type(row):
        if int(season) <= 2020 and int(row) < 17:
            return 1  # Regular season
        elif int(season) > 2020 and int(row) < 18:
            return 1
        return 0  # Playoffs

    df['regular_season'] = df['G#'].apply(game_type)
    return df

dbt Transformation Layer

Source definitions point to Snowflake's RAW_DATA schema. The staging model standardizes column names, adds computed fields, and uses dbt's {{ source() }} macro for lineage tracking.

source.ymlYAML
version: 2

sources:
  - name: NFL_DB
    schema: RAW_DATA
    tables:
    - name: vegas
      identifier: STG_RAW_BETTING_DATA
    - name: teams
      identifier: STG_RAW_TEAM_DATA
src_vegas.sqldbt / SQL
with raw_vegas as (
    SELECT * FROM {{ source('NFL_DB', 'vegas') }}
)
SELECT
    season,
    g_num         as game_number,
    regular_season as is_regular_season,
    team, opp, spread, over_under,
    spread_result, over_under_result,
    home          as is_home_game,
    points_scored, points_allowed,
    CASE
        WHEN points_scored > points_allowed THEN 'W'
        WHEN points_scored < points_allowed THEN 'L'
        ELSE 'T'
    END AS game_outcome
FROM raw_vegas
